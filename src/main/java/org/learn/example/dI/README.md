====== STEP 1: パッケージ構成の設計 ======

冒険の旅はまず、地図（構成）を用意するところから始まります。DIを理解するために、以下のようなパッケージ構成を設計しました。これにより、依存性を注入しやすい形に整理できます。

- `game`
    - このパッケージにはメインのゲームが含まれます。冒険を始めるクラスがあります。
- `game.characters`
    - RPGの登場人物（例えば戦士や魔法使いなど）に関連するクラスが含まれます。
- `game.weapons`
    - 武器やアイテムに関連するクラスが含まれます。
- `game.services`
    - キャラクターの行動や装備などを管理するためのサービスクラスが含まれます。

この構成により、それぞれのクラスが独立して役割を持ち、DIの仕組みを用いてキャラクターが武器を装備できるように設計します。
